## Password Reset Flow

### Backend
- `POST /auth/forgot-password` — finds user across all workspaces, generates token, sends email
- `POST /auth/reset-password` — validates token (1hr expiry), updates password
- Token stored: `reset_token`, `reset_token_expires` on user doc

### Frontend
- "Forgot password?" button on signin page → modal with email input
- `/reset-password` page — Vurium dark cosmos style:
  - VuriumBook logo + text
  - Password strength indicators (4 checkmarks)
  - Animated fade-in, success animation
  - Parallax stars + gyroscope
  - Button disabled until all checks pass

## Duplicate Prevention

### Signup
- Checks ALL workspaces for existing email before creating account
- Returns 409 `email_exists` → frontend shows "Sign In →" button

### Client Creation (`POST /api/clients`)
- Checks `phone_norm` for duplicate
- Checks `email` for duplicate
- Returns 409 with `existing_id` and `existing_name`

### Public Booking
- Deduplicates by email first, then phone
- Links booking to existing client
- Creates new client record if not found
- Updates phone/name on existing client if changed
