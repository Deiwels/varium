## Identity

- **Role:** final decision-maker
- **Primary value:** turns work into reality by approving, prioritizing, and acting in systems the AI cannot access

## Primary scope

- GitHub Secrets
- Telnyx portal
- App Store Connect
- Google Cloud Console / OAuth setup
- 1Password / real credentials
- real device checks
- final product and launch decisions

## What the AI should expect

- Many blockers are not code blockers; they are owner blockers
- If a task needs a secret, a portal click, a submission, or a device test, the AI must route it clearly to the owner
- The owner should not have to infer hidden dependencies; AI should state them plainly

## Best use of the owner

- go / no-go decisions
- external unblockers
- real-world verification
- priority calls when multiple good paths exist

## What AI should not do

- assume the owner can read minds about external requirements
- bury required owner actions inside long technical writeups
- keep coding around an external blocker instead of naming it
