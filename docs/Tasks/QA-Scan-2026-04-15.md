## New items from Sprint 1 changes

### [NEW-001] `SMS & 10DLC.md` stale row — Auto-provision incorrectly marked "Removed"
- **File**: `docs/Features/SMS & 10DLC.md` line 386
- **What**: Row says `Auto-provision TFN on signup | ❌ Removed — SMS via Settings now`
- **Reality**: `autoProvisionSmsOnActivation()` is implemented at `backend/index.js:2099` and called from signup (line 3821), Apple verify (line 10372), and Stripe events (line 10466)
- **Impact**: Misleads AI agents about implementation state — causes duplicate work risk
- **Fix owner**: AI 1 (docs scope)
- **Status**: NEW — fix in next docs commit

### [NEW-002] `req.workspaceId` used in apple-verify for auto-provision
- **File**: `backend/index.js` line 10372
- **What**: `autoProvisionSmsOnActivation({ id: req.workspaceId, ... }, 'apple_paid')` — but push register uses `req.wsId` (the correct middleware field). Need to verify which field apple-verify middleware sets.
- **Impact**: If `req.workspaceId` is undefined here, auto-provision for Apple IAP path silently fails — workspace ID would be empty string
- **Check**: Confirm `resolveWorkspace` middleware sets `req.workspaceId` OR `req.wsId` for the `/api/billing/apple-verify` route
- **Fix owner**: AI 1
- **Status**: NEW — needs verification

### [NEW-003] `runSmsAutoProvisionRetry` iterates ALL workspaces every 5 min
- **File**: `backend/index.js` lines 9451–9499
- **What**: Every 5-minute cycle paginates all workspaces looking for `sms_auto_provision_next_retry_at`. At scale this reads all workspace `settings/config` docs. Currently capped at `maxPerCycle = 500`.
- **Impact**: At 500+ workspaces: 500 Firestore reads every 5 minutes just for retry checks. Not a launch blocker but a future cost concern.
- **Suggested optimization**: Add a top-level `sms_provision_needs_retry: true` flag per workspace, query only those — O(flagged workspaces) not O(all workspaces).
- **Fix owner**: AI 1 (post-launch P3)
- **Status**: NEW — not a launch blocker
