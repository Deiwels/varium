# 2026-04-13 — Square Terminal Integration

> [[Home]] > DevLog | Related: [[Payments]], [[Booking System]], [[Attendance & Payroll]]

## Square OAuth Connection — Fixed
- Square OAuth connect button was not working — multiple issues found and resolved
- Added `trust proxy` to Express for correct HTTPS on Cloud Run
- OAuth callback now redirects to frontend (`/settings?tab=square&square=connected`) instead of dead-end HTML page
- Added `redirect_uri` to token exchange request (Square requires it)
- Settings page reads `tab` from URL params on mount for correct tab selection after redirect
- Added `CUSTOMERS_READ CUSTOMERS_WRITE` to OAuth scopes for customer data access
- Production keys (`SQUARE_APP_ID`, `SQUARE_APP_SECRET`) configured in GitHub Secrets

## Square Terminal Device Setup
- Added Terminal Device picker in Settings > Integrations
- Added Square Location picker (required for terminal auth)
- Backend resolves device ID: request body > workspace settings > env var > auto-fetch
- Fixed device ID format: Square Terminal API requires serial number from `device:SERIAL` format, not `device_code_id`
- Added `GET /api/square/locations` endpoint
- Devices endpoint now extracts `serial_number` and `device_code_id` from `/v2/devices` response

## Terminal Payment Flow — Fixed
- **Critical bug**: frontend sent `amount` in dollars, backend treated as cents — fixed with `amount * 100`
- **Amount vs Total**: was using `total_money` (service + tip) instead of `amount_money` (service only) — tips were double-counted as service amount
- **Tip recording**: `checkout.tip_money` is always null when `allow_tipping=true` — now fetches tip from `/v2/payments/{payment_id}` instead
- Booking marked as paid immediately on COMPLETED, tip fetched separately (non-blocking)
- Frontend polling increased to 3 min timeout, added error counting, delayed re-fetch 5s for late tips
- Added `custom_tip_field: true` and `tip_percentages` to terminal checkout for custom tip amounts
- Removed unsupported `show_receipt_screen` from device_options

## Payment Display in Calendar
- Added Amount, Tip, Total breakdown in paid booking view (was only showing "Paid via terminal")
- Tip shown in green when > $0, gray when $0

## Square Customer Sync
- Auto-sync client to Square when processing terminal payment
- Searches by phone number, creates customer if not found
- Links `square_customer_id` to local client record
- Client search fallback: when local search finds nothing, searches Square by phone and imports
- Terminal checkout note now includes: `VuriumBook - Client Name - Service - w/ Barber - Booking ID`

## Auto-Reconciliation
- Background auto-reconcile runs on every bookings load (throttled to 2 min)
- Checks Square for today's completed payments and matches to unpaid bookings
- Matching strategies: booking_id from note, then date + amount (+-$2 tolerance)
- `POST /api/payments/reconcile` — manual reconcile endpoint
- Webhook handler for `payment.completed` / `payment.updated` — auto-matches incoming payments
- `POST /api/payroll/sync-tips-from-square` — fetches tips from Square Payment API for records with tip=0

## Receipts
- Added `POST /api/receipts/send` — SMS receipt to client with service, amount, tip, total
- "Send Receipt" button in booking modal after payment

## Fees & Tax System — Fixed
- **Bug fix**: `fees` and `charges` were not saving to backend (missing from nested object handlers)
- Tax and fees now unified in single "Taxes & Fees" card in settings
- Each fee/tax has `applies_to` dropdown: All / Terminal / Cash / Zelle / Other
- `calcTotal()` filters fees AND tax by selected payment method
- Price recalculates when switching payment method in booking modal
- Payroll correctly uses `service_amount` for commission — fees/tax not included in barber payout

## Terminal Button Visibility
- Terminal payment button only shows when Square or Stripe is connected
- Calendar page fetches connection status on mount, passes `terminalEnabled` prop to BookingModal
- Cash, Zelle, Other buttons always visible

## Rate Limiting
- Increased API rate limit from 120 to 300 req/min — settings page parallel requests were hitting the limit

## Files Changed
- `backend/index.js` — Square OAuth, terminal payments, reconciliation, receipts, fees persistence
- `app/settings/page.tsx` — device/location picker, unified fees/tax UI, tab from URL
- `app/calendar/booking-modal.tsx` — terminal polling, payment display, fee filtering
- `app/calendar/page.tsx` — terminalEnabled state, connection status fetch
- `app/payments/page.tsx` — Reconcile button, Sync Tips via backend endpoint
- `lib/api.ts` — unchanged (API client)
