# Feature Flow

> Part of [[Home]] > Runbooks | See also: [[00-System/Automation-Workflow-Layer|Automation Workflow Layer]], [[Tasks/In Progress|In Progress]], [[08-Runbooks/System/Escalation-Matrix|Escalation Matrix]]

## Purpose

Define the standard path for feature requests through framing, research, planning, implementation, QA, and Owner verification.

## Step-by-step Procedure

1. Classify trivial vs non-trivial.
2. For non-trivial work, route to AI 6 first.
3. Route to AI 5 if external truth is required.
4. Route to AI 7 if compliance/system translation is required.
5. Route to AI 3 for the execution plan.
6. Run the review gate.
7. Route implementation to AI 1 / AI 2.
8. Run AI 3 QA.
9. Wait for Owner live verification.
