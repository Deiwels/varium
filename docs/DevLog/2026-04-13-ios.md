## iOS App — Calendar & Dashboard Fixes

### Calendar date dots not visible in iOS app
- **Root cause**: `.content` container has `overflow: auto` + `position: relative; z-index: 1` creating a stacking context. Fixed elements inside it (date dots, gear/plus buttons) were trapped below pill-bar (z-index 9999 injected by iOS native code)
- **Fix**: Portal date dots and gear/plus buttons to `document.body` via `createPortal`
- Date dots z-index: 61 -> 10001, gear/plus: 9999 -> 10002
- Different `bottom` values for native (74px) vs web (50px) using `window.__VURIUM_IS_NATIVE`

### Dashboard pagination (iPhone home screen style)
- `availH` calculation now subtracts pill-bar height (56px) + safe area bottom
- Items that don't fit overflow to next page with swipe navigation
- Page dots: `position: fixed`, `bottom: 80px`, `z-index: 10001`
- Page dots dimmer than before (opacity .45/.15) per user preference

### Dashboard jiggle mode (edit widgets)
- Done button (left) + Add button (right) portaled to `document.body`
- Uses `--shell-top` CSS variable for positioning below header
- z-index 99999, semi-transparent backdrop blur background
- Add Widget sheet expanded to maxHeight 70%, padding-bottom 80px

### Schedule widget redesigned
- Two-column layout showing free slots per barber
- Shows next 3 available 30-min slots, filters past times and off-day barbers
- Compact height (minHeight 55px)

### Cookie banner hidden in native app
- `CookieBanner.tsx`: check `window.__VURIUM_IS_NATIVE` to skip rendering

### Booking modal scroll fix
- Block background scroll when modal open (`overflow: hidden` on body + `.content`)
- `onTouchMove` stopPropagation prevents calendar scrolling behind modal
- Modal uses `alignItems: flex-start` with scroll on outer container

## Push Notifications — Full Implementation

### Bundle ID fix (critical)
- Backend had hardcoded `com.vuriumbook.app`, iOS app uses `com.vurium.VuriumBook`
- Changed default + added `APNS_BUNDLE_ID` env var support

### APNs environment fix
- Default changed from `production` to `sandbox` for dev builds
- Added support for `development` env value (maps to sandbox endpoint)

### APNs key format fix
- P8 key stored as base64 in GitHub Secrets
- Added base64 decode in `getApnsJwt()` function
- Added diagnostic endpoint `/api/push/status` for debugging

### Push token isolation per workspace
- On `/api/push/register`: remove device token from ALL other workspaces first
- Added `/api/push/unregister` endpoint for logout
- Auto re-register push token on every Shell load (switches workspace correctly)
- On Sign Out: call unregister + clear native UserDefaults via webkit message handler

### Push notification preferences enforcement
- Settings -> Booking toggles now actually control push sending
- Preferences read from workspace settings (not per-user which was never set)
- Each push type checks workspace preference before sending:
  - `push_confirm` -> booking confirmation
  - `push_cancel` -> cancellation
  - `push_reschedule` -> reschedule
  - `push_waitlist` -> waitlist
  - `push_chat_messages` -> chat messages
- Removed 24h/2h reminder toggles from UI (not implemented on backend)

### Push routing — only to assigned barber
- `sendCrmPushToStaff` no longer broadcasts to all owners/admins
- Booking push only sent to the specific barber the appointment is for
- User who created booking excluded from receiving push (`excludeUserId`)

### iOS foreground push behavior
- Chat messages: show banner + sound (always)
- Booking confirmations: hidden when app active (user created it)
- Other: banner only, no sound

### Push token persistence
- iOS app now restores auth token from UserDefaults to localStorage on WebView load
- Prevents logout on app restart when iOS clears localStorage
- Native logout handler clears UserDefaults via webkit message

### Error logging
- All push calls now log errors instead of silently swallowing
- Startup check logs whether APNs is configured or disabled
- Response status logging for failed APNs calls
