## Developer Panel — New Feature (`/developer`)
Private admin panel for vurium.com with separate auth system.

### Auth — Magic Link
- Separate from VuriumBook auth (no VuriumBook account needed)
- Enter email on `/developer/login` → receive magic link via Resend → click → HttpOnly cookie session (24h)
- Rate limited: 3 requests per 15 min per IP
- One-time use tokens (jti + Set)
- IP logging on every login (Firestore `vurium_dev_logins`)
- Cookie: `vurium_admin_token`, HttpOnly, Secure, SameSite=none (for cross-origin vurium.com ↔ run.app)
- Env var: `ADMIN_EMAIL` controls who receives magic link

### Overview Page (`/developer`)
- Workspace count (total, paid, trialing)
- Trial conversion rate
- Signups (7d, 30d) + signup chart
- Plan breakdown (Individual / Salon / Custom)
- Platform totals: bookings, clients, staff across all workspaces
- Workspaces table: name, slug, plan, billing status, bookings, clients, staff, SMS status, created date
- Backend: `GET /api/vurium-dev/platform` aggregates from all Firestore workspace docs

### Analytics Page (`/developer/analytics`)
- Custom privacy-friendly tracker (`public/va.js`, ~1KB)
- Tracks: page URL, referrer, screen size, device type, browser
- Visitor ID via localStorage (not cookie), session ID via sessionStorage
- IP hashed with SHA-256 before storage
- SPA-aware: hooks into pushState/replaceState/popstate
- Backend: `POST /t` (unauthenticated, rate-limited 60/min per IP)
- Dashboard: pageviews by day (SVG chart), unique visitors, sessions, pages/session
- Top pages, top referrers, device breakdown, browser breakdown
- Conversion funnel: landing → signup → dashboard
- Range filter: 7d / 30d / 90d
- Firestore collection: `vurium_analytics`

### Email Page (`/developer/email`)
- Gmail mailbox quick links (support@, billing@, sales@, security@vurium.com)
- Branded compose: write text → sends via Resend with Vurium dark cosmos HTML template from noreply@vurium.com
- Sent history from Firestore `vurium_emails`
- Gmail API integration (frontend ready, backend endpoints needed — see next session)
- Inbound email forwarding: when email arrives, notification with preview sent to ADMIN_NOTIFY_EMAIL

### Backend Endpoints (all under `requireSuperadmin` middleware)
- `POST /api/vurium-dev/auth/request` — magic link request
- `POST /api/vurium-dev/auth/verify` — verify magic link → set cookie
- `POST /api/vurium-dev/auth/logout` — clear cookie
- `GET /api/vurium-dev/ping` — auth check
- `GET /api/vurium-dev/platform` — workspace/business metrics
- `GET /api/vurium-dev/analytics` — site analytics aggregation
- `POST /api/vurium-dev/email/send` — send branded email
- `POST /api/vurium-dev/email/inbound` — Resend inbound webhook
- `GET /api/vurium-dev/emails` — list emails
- `GET /api/vurium-dev/emails/:id` — get single email
- `PATCH /api/vurium-dev/emails/:id` — mark read/unread
- `POST /t` — analytics tracker ingest (unauthenticated)
