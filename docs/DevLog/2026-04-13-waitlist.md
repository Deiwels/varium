### Email Notifications (Styled per Template)
- **Confirmation email** ‚Äî sent when client joins waitlist
  - Subject: "You're on the waitlist ‚Äì {Shop Name}"
  - Content: date, preferred time, service, duration in card layout
  - Uses `vuriumEmailTemplate()` + `EMAIL_THEMES` + workspace `site_config.template`

### Resolve Endpoint ‚Äî `waitlist_enabled`
- `GET /public/resolve/:slugOrId` now returns `waitlist_enabled: true/false`
- Derived from `getPlanDef(effectivePlan).features.includes('waitlist')`
- salon/custom/trial = true, individual/expired = false

### Waitlist Role Permissions
- New `waitlist` category in `PermissionsProvider.tsx` with 4 keys:
  - `view_ghost` ‚Äî see breathing waitlist blocks on calendar
  - `confirm` ‚Äî confirm/book/remove from waitlist
  - `view_phone` ‚Äî see waitlist client phone number
  - `call_client` ‚Äî call waitlist client directly
- Defaults: admin=all on, barber=view_ghost only, guest/student=all off
- Added to Settings ‚Üí Permissions tab (`PERM_SECTIONS`)
- Applied in: calendar ghost blocks, confirm modal (phone + call button), waitlist page (phone, confirm/remove buttons)

### Admin Waitlist Page Fixes
- Added `preferred_start_min`/`preferred_end_min` to TypeScript `WaitlistEntry` interface
- Removed `as any` casts
- Phone display + call button gated by `waitlist.view_phone` / `waitlist.call_client` permissions
- Confirm/Remove buttons gated by `waitlist.confirm` permission

### Calendar Integration (Already Existed ‚Äî Connected)

- Searches within `preferred_start_min` ‚Äì `preferred_end_min` window
