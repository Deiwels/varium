# Handoffs

> Part of [[Home]] > Tasks | See also: [[00-System/Automation-Workflow-Layer|Automation Workflow Layer]], [[04-Tasks/Workflow-Queue|Workflow Queue]], [[11-Reference/Note-Templates|Note Templates]]

## Purpose

Provide one canonical handoff standard so work can move between lanes without vague status or hidden assumptions.

## Required Fields

- current status
- what was completed
- what remains
- canonical notes to read
- unresolved questions
- next owner
- blocking or non-blocking
- recommended next action

## Bad Handoffs

- `done`
- `check notes`
- `should be fine`

## Good Handoff Shape

`Growth brief complete in [[Growth/Growth-Backlog|Growth Backlog]]. Product claim boundary verified against [[Product/Feature-Briefs/Reminder-SMS-Launch-Readiness|Reminder SMS Launch Readiness]]. One compliance wording check remains before creative can ship. Next owner: AI 7. Status: blocked pending wording confirmation.`
