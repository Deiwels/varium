## Retrospective: Sprint 1 Regressions (for future scans)

### Waitlist regression (2026-04-15) — SET-006 fix caused CTA disappearance

**What happened:**
- Commit `849e998` (SET-006 fix) forced `waitlist_enabled: false` in `/public/config/` for every workspace that never touched the Settings toggle
- Frontend `??` fallback couldn't kick in because `false` is not nullish
- Result: `Join waitlist` CTA disappeared from public booking page

**Root cause:**
- Backend wrote explicit `false` instead of leaving field undefined
- Frontend relied on nullish coalescing for default-on behavior

**Fix:**
- Commit `a3c885f` — changed to conditional spread so undefined falls through to plan feature check
- Semantic: waitlist is ON by default on any plan that includes it; Settings toggle is explicit OFF override

**Lesson:**
- When changing default behavior, check both backend write sites AND frontend fallback logic
- Explicit `false` in Firestore blocks frontend `??` — use `undefined` or omit field for "use default"

**Pattern to watch for in future scans:**
- Any `field: false` write that replaces `field: undefined` or field omission
- Any boolean field where `false` should mean "default" rather than "explicitly off"
