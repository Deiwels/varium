## New Feature: History Page (`/history`)

Full booking + waitlist history view with filters, stats, and Varium Dark Cosmos styling.

### What It Does
- Unified history of **bookings** and **waitlist** entries in one list
- Stats row: total records, completed, revenue, waitlist count
- Filters: search (client/service/barber), type (bookings/waitlist), date range (today/week/month/all), status, barber, sort direction
- Waitlist entries fetched from `GET /api/waitlist`, bookings from `GET /api/bookings?limit=1000`
- Waitlist items have gold color indicator; bookings use barber color
- Status chips: `waitlist`, `waitlist-confirmed`, `waitlist-removed` for waitlist; standard booking statuses for bookings

### Navigation Changes
- **Bottom pill nav**: removed Clients, reordered to Home | History | **Calendar** (center) | Messages | Settings
- **Sidebar NAV**: added History after Calendar (visible to all roles)
- **Dashboard shortcuts**: added History icon in both mobile (`iconSvgs`) and desktop (`shortcutIcons`) grids
- Icon: document with 3 horizontal lines (list view)

### Files Created
- `app/history/page.tsx` — full page component with Shell wrapper

### Files Modified
- `components/Shell.tsx` — added `history` to NAV array, Icon component, and bottom pill bar
- `app/dashboard/page.tsx` — added History to `allActions`, `iconSvgs` (mobile), `shortcutIcons` (desktop)

### Technical Notes
- Dashboard has **two separate icon maps**: `iconSvgs` (mobile, keyed by label) and `shortcutIcons` (desktop, keyed by href). Both need to be updated when adding new shortcuts.
- Dashboard saves layout in `localStorage` (`HOME_LAYOUT_KEY`). New items appear automatically via `defaultOrder` which includes all `actions`.
- Shell `page` prop is required — must pass `page="history"` to Shell.
- Waitlist API returns `{ waitlist: [...] }` and may fail if plan doesn't include waitlist feature — handled with `.catch(() => ({ waitlist: [] }))`.
