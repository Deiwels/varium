## DM Messages ‚Äî Critical Fix

### Problem
Direct messages between employees were broken. When Dasha sent a DM to Nazarii, the chat appeared empty on Nazarii's side.

### Root Cause
1. **Empty `uid` in localStorage** ‚Äî `auth/login` returned `{ id: ... }` but NOT `uid`. Frontend code used `user?.uid` which was `undefined`.
2. **Broken chatType** ‚Äî `dmChatType('', recipientId)` produced `dm__recipientId` (double underscore) instead of `dm_senderId_recipientId`.
3. **ID mismatch between roles** ‚Äî Owner/admin loaded staff from `/api/users` (user IDs), barbers fell back to `/api/barbers` (barber IDs). Two users would generate different chatType for the same conversation.

### Fixes
- `auth/login` now returns both `id` and `uid` (`uid: userDoc.id`) ‚Äî matches `auth/login-email`
- Frontend: `uid = user?.uid || user?.id || ''` ‚Äî fallback to `id`
- New `GET /api/staff` endpoint ‚Äî lightweight staff list accessible to all roles, returns user collection IDs (not barber IDs)
- Frontend uses `/api/staff` instead of `/api/users` ‚Üí `/api/barbers` fallback
- Migration endpoint `POST /api/messages/fix-dm` ‚Äî fixes broken `dm__*` chatTypes by reconstructing correct `dm_senderId_recipientId` from `sender_id` field. Ran once, fixed 2 messages.

## Chat Layout ‚Äî Input Always Visible

### Problem
On both mobile and desktop, the message input field was not visible without scrolling. On iOS, the keyboard caused the input to jump and the page to bounce.

### Root Cause
- Messages container used `height: 100dvh` inside Shell's `.content` div which already handled its own height via `flex: 1`
- `.content` had `overflow: auto` + `padding-bottom: 56px` for bottom nav ‚Äî container overflowed and became scrollable
- iOS `visualViewport` handler used `window.scrollTo(0,0)` on every resize event, causing jitter

### Fixes
- Container: `height: 100%` instead of `100dvh` ‚Äî fills parent naturally
- `.content` override: `overflow: hidden` ‚Äî prevents page scroll on messages page
- iOS keyboard: rewritten with `requestAnimationFrame` + `translateY` instead of `scrollTo`
- `position: fixed` on html/body when messages page active ‚Äî prevents iOS Safari bounce
- `touchmove` blocked on container (except `.msg-list`) ‚Äî stops overscroll during keyboard
- Input `fontSize: 16px` ‚Äî prevents iOS auto-zoom on focus (< 16px triggers zoom)
- `will-change: height, transform` on container for GPU-accelerated transitions

### Commits
- `7bedeed` fix: keep content padding-bottom so chat input sits above bottom nav
- `be1bfe3` fix: iOS keyboard handling ‚Äî prevent scroll jitter and page bounce

## DM Preview in Chat List

### Feature
Each DM contact now shows the last message and relative time (like Instagram).

- Shows "You: text" for own messages, just "text" for received
- Relative time: "now", "3m", "2h", "Apr 4"
- "No messages yet" for contacts without DM history
- New endpoint `GET /api/messages/dm-previews` returns last message per DM conversation
- Preview updates locally after sending a message
